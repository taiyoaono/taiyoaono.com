ただし、このループには
止まるリスクがある。

ファンは買った直後は必ずタッチする。問題は2回目以降。

「またかざしたくなる理由」が無ければ、
アクスタはただの置物に戻る。

LTM（ライフタイムメディア）が成立するか否かは、リピートタッチ率で決まる。
つまりこの事業の生命線は「ループを回し続ける仕組み」にある。

Key Variables

ループを回し続ける
3つの変数。

1

コンテンツの鮮度

「新しいものがある」と思えるか。更新が止まった瞬間、タッチの動機が消える。

2

体験の関連性

自分に関係あるコンテンツが出てくるか。全員同じ体験では飽きが早い。

3

運用の持続性

IPが増えても回る仕組みがあるか。人手運用は3タイトル超えたら破綻する。

この3変数をソフトウェアとAIで支えるのが、2WINSの役割です。

5 / 14
変数 1 + 2 鮮度 × 関連性

データ基盤 + 配信エンジン

全タッチの行動データをリアルタイム収集・蓄積
→ 誰が・いつ・何回・何秒見たかを構造化データとして保持
ユーザーごとにコンテンツと広告を出し分ける配信制御
→ 全員同じ体験から「あなただけの体験」へ。リピート動機を作る
タッチ率が最も高い曜日・時間帯を分析し、配信タイミングを最適化
→ 新コンテンツの公開タイミングで初動タッチ率が変わる
A/Bテスト基盤でコンテンツ構成の効果を検証
→ 勘ではなくデータで「何が効くか」を回す
6 / 14
変数 1 鮮度

コンテンツ制作支援AI

更新頻度はこの事業の生命線。だが、毎回の制作にはコストがかかる。

既存素材から背景・テロップ・レイアウト違いのバリエーションを自動生成
→ 1回の撮影素材から複数コンテンツを作り、更新頻度を上げる
字幕・テロップの自動付与、配信フォーマットへの自動変換
→ 編集工数を削減し、制作→公開のリードタイムを短縮
制作進行の管理ダッシュボード
→ 複数IP並行時に「どのタイトルの更新が滞っているか」を一目で把握
7 / 14
変数 3 持続性

運用自動化

1タイトルなら人手で回せる。
10タイトルになったとき、仕組みが無ければ破綻する。

パートナー向けレポートの自動生成・定期配信
→ タイトルが増えるたびにレポート作成の手間が増えるのを防ぐ
NFC個体のライフサイクル管理（出荷→アクティベーション→利用状況）
→ 数万〜数十万のNFC個体を一元管理する基盤
キャンペーンの自動設定（期間限定コンテンツの出し入れ）
→ 設定ミスや対応漏れのリスクを仕組みで防ぐ

PoC Design

PoCで検証すべきこと。

最初の1タイトルで、以下を明らかにする。

ユーザー行動
  • 購入後、何日目に初回タッチが起きるか
  • 週あたり何回タッチされるか（減衰カーブ）
  • コンテンツ更新後、タッチ率はどれだけ回復するか
事業性
  • コンテンツ1本あたりの制作コスト
  • AI支援でどこまでコストを下げられるか
  • 出し分けの有無でエンゲージメント差が出るか

最初から大きく張るのではなく、仮説を検証して数字で語れる状態を作る。
その数字があれば、次のIPへの横展開もスポンサーへの営業も格段にやりやすくなる。
